**目标：** 为 Veloxa 引擎规划首个完整 DevTool 主线（Inspector + Performance Overlay + Hot Reload），产出蓝图级 spec + plan + 2 篇 creative，build 由用户基于本 plan 拆出独立 Level 3 子任务（Phase A / B / C 各一）。

**技术栈：** C++17 / CMake / SDL2 2.0.20（已有）/ Linux inotify（一期，~150-200 LOC 自实现）/ QuickJS（已有）/ GTest

**复杂度级别：** Level 4（多子系统蓝图 + 8 决策矩阵 + 8 威胁面）

**任务 ID：** TASK-20260430-04

**分支：** `feature/TASK-20260430-04-ui-editor-debugger`（已基于 main `2445990` 创建）

**关联文档：**
- spec：[`docs/specs/2026-04-30-devtool-design.md`](../specs/2026-04-30-devtool-design.md)

**回答 6 项必填问题：**

#### Q1. 依赖图

新增静态库 `vx_devtool`，下游目标：
- `tests/devtool/inspector/inspector_data_test.cc` 等所有 DevTool 单测
- `examples/hello_devtool.cc`（需 `VX_BUILD_DEVTOOL=ON`）

`vx_devtool` 上游依赖：

- `vx_script`（QuickJS DomBindings — DevTool UI 的 JS 通过此调 C API）
- `vx_platform`（SDL2 / inotify — file_watcher_inotify 用 sys/inotify.h）
- `vx_api`（C API 头 — DevTool 调 vx_view_serialize_dom_json 等）

#### Q2. 头文件暴露

每个下游目标的源文件 include 上游头：
- `inspector_data.cc` → `<veloxa/core/dom/serializer.h>` / `<veloxa/core/layout/layout_box.h>` ✓
- `file_watcher_inotify.cc` → `<sys/inotify.h>` / `<unistd.h>` （系统头，不需 CMake link）
- `hot_reload_manager.cc` → `<veloxa/core/application.h>` ✓
- `inspector_overlay.cc` → `<veloxa/core/render/renderer.h>` ✓

#### Q3. 符号依赖

下游目标间接依赖上游符号：
- `inspector_data.cc` 调 `vx::dom::Serialize(node)` → 实现在 `vx_core` 内 ✓
- `inspector_overlay.cc` 调 `vx::render::Renderer::AppendOverlayCommand()` → 实现在 `vx_core` 内 ✓
- `file_watcher_inotify.cc` 调 `inotify_init1` / `inotify_add_watch` → libc 系统调用，无 CMake link

#### Q4. 当前链接方向

`vx_core` 当前对外（PUBLIC）暴露：
- `<veloxa/core/dom/...>` 头 PUBLIC ✓
- `<veloxa/core/layout/...>` 头 PUBLIC ✓
- `<veloxa/core/render/...>` 头 PUBLIC ✓
- `<veloxa/core/event/...>` 头 PUBLIC ✓

`vx_script`：DomBindings 头需 PUBLIC（DevTool UI 的 JS 通过 DomBindings 调 C API）— 待 grep 验证（一期可能仍 PRIVATE，需提升为 PUBLIC）

#### Q5. 结论

`vx_devtool` CMakeLists.txt 链接：

```cmake
add_library(vx_devtool STATIC
  inspector/inspector_data.cc
  inspector/inspector_overlay.cc
  overlay/perf_overlay.cc
  hot_reload/file_watcher.cc
  hot_reload/file_watcher_inotify.cc
  hot_reload/hot_reload_manager.cc
)
target_include_directories(vx_devtool PUBLIC ${PROJECT_SOURCE_DIR})
target_link_libraries(vx_devtool
  PUBLIC vx_core vx_script vx_platform   # 头需 PUBLIC，下游 test / example 复用
  PRIVATE Threads::Threads                # std::thread for inotify watch loop
)

# DevTool 资源（HTML/CSS/JS）打包为 binary blob
# (可选 — 一期可走 runtime 文件读取路径，避免 binary blob 工具链)

option(VX_BUILD_DEVTOOL "Build DevTool subsystem" OFF)  # 默认 OFF
if(VX_BUILD_DEVTOOL)
  add_subdirectory(veloxa/devtool)
endif()
```

如果 vx_script 当前 DomBindings 头 PRIVATE：在 Phase A.0 前置任务加 `target_link_libraries(vx_script PUBLIC quickjs)` 升级。

### 任务 0.7：调用链端到端验证（writing-plans §调用链端到端验证必填，I1 改造关键）

#### I1 Application 双 Document 槽改造完整调用链

从最上游入口到最下游被改字段：

| 层 | 文件 + 函数 | 当前 | 改后 | 透传？ |
|:-:|---|---|---|:-:|
| 1 | `vx_view_create()` `veloxa_api.cc` | 创建单 Document | 创建 target_document，devtool 槽 = nullptr | — |
| 2 | `Application::ctor` | `document_ = ...` | `target_document_ = ...; devtool_document_ = nullptr;` | ✅ 重命名透传 |
| 3 | `Application::document()` getter | `return *document_;` | 移除（callsite 改用 target_document() / devtool_document()）| ✅ 全 callsite 改 |
| 4 | `Application::Update()` | `document_.Update()` | `target_document_.Update(); if(devtool_document_) devtool_document_->Update();` | — |
| 5 | `EventManager::dispatch` | 持 `Document*` | 继续持 target_document_ 的引用（DevTool 单独 EventManager 实例）| ✅ 改构造参数 |
| 6 | `UpdateManager::Update` | 持 `Document&` | 同 5 | ✅ |
| 7 | DOM bindings | `app->document()` | `app->target_document()` | ✅ |

**透传决策：** 第 3 层 `document()` getter 移除是核心 — 所有 callsite 必须改 → 编译期暴露漏改（R1 mitigation）。
