## Phase 1：Document ArenaAllocator

**目标：** Document 节点分配从 `new`/`delete` 迁移到 ArenaAllocator，提升缓存局部性和减少 malloc 压力。

### 设计要点

- `Document` 持有 `ArenaAllocator arena_`
- `CreateElement`/`CreateText`/`CreateComment` 使用 `arena_.Allocate` + placement new
- `owned_nodes_` 保留，用于在 Document 析构时调用 `node->~Node()`
- 析构顺序关键：先遍历 `owned_nodes_` 调用虚析构（释放 String/SmallVector 的堆内存），然后 `arena_` 析构释放所有节点对象内存
- Document 自身不在 Arena 中（顶层对象）
- Node 已有 `virtual ~Node() = default;`，派生类析构函数正确释放成员

### 1.1 修改 `document.h`

```cpp
// 新增 include
#include "veloxa/foundation/memory/arena_allocator.h"

class Document : public Element {
 public:
  explicit Document(usize arena_block_size = 4096)
      : Element(TagId::kUnknown, NodeType::kDocument),
        arena_(arena_block_size) {}

  ~Document() override {
    for (auto* node : owned_nodes_) {
      node->~Node();
    }
    // arena_ 在此之后析构，批量释放所有节点内存
  }

  Element* CreateElement(TagId tag_id);
  Text* CreateText(String data);
  Comment* CreateComment(String data);

  usize arena_bytes_allocated() const { return arena_.bytes_allocated(); }

 private:
  ArenaAllocator arena_;
  Vector<Node*> owned_nodes_;
};
```

### 1.2 修改 `node.cc` Document 方法

```cpp
Element* Document::CreateElement(TagId tag_id) {
  void* mem = arena_.Allocate(sizeof(Element), alignof(Element));
  auto* el = new (mem) Element(tag_id);
  owned_nodes_.push_back(el);
  return el;
}

Text* Document::CreateText(String data) {
  void* mem = arena_.Allocate(sizeof(Text), alignof(Text));
  auto* text = new (mem) Text(std::move(data));
  owned_nodes_.push_back(text);
  return text;
}

Comment* Document::CreateComment(String data) {
  void* mem = arena_.Allocate(sizeof(Comment), alignof(Comment));
  auto* comment = new (mem) Comment(std::move(data));
  owned_nodes_.push_back(comment);
  return comment;
}
```

## Phase 2：ID 查找优化 (O(N) → O(1))

**目标：** `TagIdFromName`（68 条目）和 `PropertyIdFromName`（57 条目）从线性扫描升级为 HashMap 查找。

### 设计要点

- 使用 `vx::HashMap<StringView, TagId/PropertyId, StringViewHash, StringViewEq>`
- 懒初始化（Meyer's singleton：static local + lambda）— 避免静态初始化顺序问题
- `StringViewHash`/`StringViewEq` 定义在 `.cc` 文件的匿名命名空间中
- TagIdFromName 需大小写不敏感：栈上 lowercase 副本后查 HashMap
- PropertyIdFromName 直接查找（CSS 属性名已由 tokenizer 保证小写）
- 调用 `reserve()` 预分配避免多次 rehash

### 2.1 修改 `tag.cc`

```cpp
#include "veloxa/foundation/containers/hash_map.h"

namespace {

// ... kTagTable[] 保持不变 ...

struct StringViewHash {
  usize operator()(StringView sv) const {
    usize hash = 5381;
    for (usize i = 0; i < sv.size(); ++i) {
      hash = ((hash << 5) + hash) + static_cast<u8>(sv[i]);
    }
    return hash;
  }
};

struct StringViewEq {
  bool operator()(StringView a, StringView b) const {
    return a == b;
  }
};

using TagMap = HashMap<StringView, TagId, StringViewHash, StringViewEq>;

TagMap* BuildTagMap() {
  auto* m = new TagMap();
  m->reserve(kTagTableSize);
  for (usize i = 1; i < kTagTableSize; ++i) {
    m->Insert(StringView(kTagTable[i].name), kTagTable[i].id);
  }
  return m;
}

}  // namespace

TagId TagIdFromName(StringView name) {
  static TagMap* map = BuildTagMap();

  char buf[64];
  usize len = name.size();
  if (len == 0 || len >= sizeof(buf)) return TagId::kUnknown;
  for (usize i = 0; i < len; ++i) {
    char c = name[i];
    buf[i] = (c >= 'A' && c <= 'Z') ? static_cast<char>(c + 32) : c;
  }

  TagId* result = map->Find(StringView(buf, len));
  return result ? *result : TagId::kUnknown;
}
```

删除 `CaseInsensitiveEqual` 辅助函数（不再需要）。
